MESSAGE FLOW DIAGRAM:
═══════════════════════════════════════════════════════════════════════════════

1. Alice sends message:
   ↓
2. Server creates message_id (e.g., 1)
   ↓
3. Server stores: messages[1] = {sender: alice, room: general, ...}
   ↓
4. Server marks delivered_to: [bob, charlie]
   ↓
5. Server broadcasts: "alice: hello\nMSG_ID:1\n"
   ↓
6. Bob receives and sees MSG_ID:1
   ↓
7. Bob sends /ack 1
   ↓
8. Server marks read_by: [bob]
   ↓
9. Message shows [✓✓] read status
