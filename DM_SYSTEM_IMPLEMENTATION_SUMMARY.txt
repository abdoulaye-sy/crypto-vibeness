2. HELPER FUNCTION: get_dm_session_key (Lines 376-379)

    def get_dm_session_key(user1, user2):
        """Create a consistent session key for DM"""
        users = sorted([user1.lower(), user2.lower()])
        return f"{users[0]}:{users[1]}"

   Logic: Alphabetical sort ensures "alice:bob" and "bob:alice" → same key
   Thread-safe: Runs synchronously, no race conditions

3. HELPER FUNCTION: send_dm (Lines 382-389)

    def send_dm(sender, recipient, message):
        """Send a DM message"""
        key = get_dm_session_key(sender, recipient)
        with dm_lock:
            if key not in dm_sessions:
                dm_sessions[key] = []
            dm_sessions[key].append({
                'sender': sender,
                'text': message,
                'timestamp': time.time()
            })

   Lock: Protects dm_sessions dict
   Storage: Each message as dict with metadata

4. HELPER FUNCTION: find_recipient_socket (Lines 392-398)

    def find_recipient_socket(recipient_username):
        """Find the socket of a connected user"""
        with clients_lock:
            for sock, client_info in clients.items():
                if client_info['username'].lower() == recipient_username.lower():
                    return sock
        return None

   Lock: Uses clients_lock (correct lock ordering)
   Match: Case-insensitive username comparison
   Return: Socket object or None if not found

THREAD SAFETY ANALYSIS
═══════════════════════════════════════════════════════════════════════════════

Lock Dependencies:
─────────────────

New Lock: dm_lock (Thread level: 5)
- Protects: dm_sessions dict

Existing Locks Used in DM Code:
- clients_lock (Level 4): Used in find_recipient_socket()
- reset_tokens_lock: Not used

Lock Ordering (Enforced):
  Level 1: typing_lock
  Level 2: messages_lock
  Level 3: rooms_lock
  Level 4: clients_lock ← find_recipient_socket() uses this
  Level 5: reset_tokens_lock
  
  Independent: dm_lock (only protects dm_sessions)

✓ NO DEADLOCK RISK: find_recipient_socket acquires clients_lock,
  which is at correct hierarchy level.

✓ RACE CONDITIONS PREVENTED:
  - send_dm() has dm_lock
  - find_recipient_socket() has clients_lock
  - Database operations are atomic (load_user_database)
