TECHNICAL ARCHITECTURE:

Data Structures:
  rooms = {'general': [sock1, sock2], 'crypto': [sock3]}
  clients = {sock1: {username, room, address}}
  reset_tokens = {token: (username, expiry_time)}

Thread Safety:
  - rooms_lock: Protects rooms dict
  - clients_lock: Protects clients dict
  - reset_tokens_lock: Protects tokens dict
  - Lock ordering: rooms → clients → tokens (prevents deadlock)

Message Flow:
  1. Client sends message
  2. Server checks if starts with "/" (command)
  3. If command: handle_chat_command()
  4. If regular: broadcast_to_room() current room only
