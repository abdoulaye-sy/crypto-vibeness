===============================================================================
MIGRATION MD5 → BCRYPT - DOCUMENTATION DES CHANGEMENTS
===============================================================================

OBJECTIF:
Remplacer le hachage MD5 (insecure) par bcrypt pour sécuriser les mots de passe.

===============================================================================
CHANGEMENTS EFFECTUÉS
===============================================================================

1. IMPORT BCRYPT
   - Ajout: import bcrypt
   - Suppression: import hashlib (non utilisé après)

2. CONFIGURATION BCRYPT
   - BCRYPT_ROUNDS = 12 (coût computationnel - plus haut = plus lent mais sécurisé)

3. FONCTION hash_password() - REMPLACÉE
   ❌ AVANT (MD5 - INSECURE):
      def hash_password(password):
          return hashlib.md5(password.encode()).hexdigest()
      
      Problèmes:
      - MD5 est rapide (attaque par force brute simple)
      - Pas de salt (hashes identiques pour mêmes mots de passe)
      - Vulnérable aux rainbow tables

   ✅ APRÈS (BCRYPT - SECURE):
      def hash_password(password):
          salt = bcrypt.gensalt(rounds=BCRYPT_ROUNDS)
          hashed = bcrypt.hashpw(password.encode('utf-8'), salt)
          return hashed.decode('utf-8')
      
      Avantages:
      - Bcrypt inclut un salt aléatoire dans le hash
      - Coût adaptatif (BCRYPT_ROUNDS = 12)
      - Résistant aux attaques brute-force modernes
      - Hash unique pour chaque mot de passe (même identique)

4. FONCTION verify_password() - NOUVELLE
   ✅ AJOUTÉE:
      def verify_password(password, hashed_password):
          return bcrypt.checkpw(password.encode('utf-8'), 
                               hashed_password.encode('utf-8'))
      
      Avantages:
      - Utilise bcrypt.checkpw() pour vérifier correctement
      - Résistant aux timing attacks
      - Gestion d'erreurs intégrée

5. FONCTION authenticate_user() - MODIFIÉE
   ❌ AVANT:
      password_hash = hash_password(password)
      if password_hash == stored_hash:  # Comparaison directe (dangereux)
          return 'AUTH_SUCCESS', stored_username

   ✅ APRÈS:
      if verify_password(password, stored_hash):  # Utilise bcrypt
          return 'AUTH_SUCCESS', stored_username

===============================================================================
FORMAT DES HASHES STOCKÉS
===============================================================================

MD5 (ancien):
  Format: 32 caractères hexadécimaux
  Exemple: 35b95f7c0f63631c453220fb2a86f218
  Pas de salt, pas d'aléa

BCRYPT (nouveau):
  Format: $2b$ROUNDS$22_CHARS_SALT$31_CHARS_HASH
  Exemple: $2b$12$H7kzlHXVPTOSFd6h96O/2..MF1LwXtWP4qD14Rb7k7Kjv48KTKJR2
  - Longueur: 60 caractères
  - Salt inclus dans le hash
  - Aléatoire à chaque hash

===============================================================================
SÉCURITÉ BCRYPT vs MD5
===============================================================================

Temps de cracking (estimé pour 1 GPU moderne):

MD5:
  - 8 caractères: quelques minutes
  - 12 caractères: quelques heures
  - 14 caractères: quelques jours

BCRYPT (rounds=12):
  - 8 caractères: plusieurs jours
  - 12 caractères: plusieurs semaines/mois
  - 14 caractères: plusieurs mois/années
  - Devient plus difficile à mesure que les ordinateurs s'améliorent (coût adaptatif)
