COMMAND INTERCEPTION FIX - SUMMARY
═══════════════════════════════════════════════════════════════════════════════

PROBLEM:
--------
Commands like /rooms, /join, /leave were being treated as regular chat messages
and broadcast to all room members instead of being intercepted and processed.

Example: When user typed "/rooms", the server would broadcast:
  [user]: /rooms
Instead of executing the command and returning the room list.

ROOT CAUSE:
-----------
In the main chat loop (handle_client function), the condition checking for
commands was incomplete:

  OLD CODE (broken):
  if message.startswith('/'):
      cmd_result, new_room = handle_chat_command(...)
      if new_room:              # BUG: new_room is None for successful commands
          current_room = new_room
  else:
      broadcast_to_room(...)

The problem: handle_chat_command() returns (None, current_room) for successful
commands, so `if new_room:` would be False, and the room wouldn't update.

SOLUTION:
---------
Changed the condition to explicitly check for None:

  NEW CODE (fixed):
  if message.startswith('/'):
      print(f"[CMD] {message} received from {username}")
      cmd_result, new_room = handle_chat_command(...)
      if new_room is not None:  # FIX: Check for None explicitly
          current_room = new_room
          print(f"[ROOM_UPDATE] {username} now in {current_room}")
  else:
      broadcast_to_room(f"{username}: {message}\n", current_room)

CHANGES MADE:
-------------
1. Added explicit check: `if new_room is not None:` instead of `if new_room:`
2. Added debug logging: `[CMD]` message when command received
3. Added debug logging: `[ROOM_UPDATE]` when room changes
4. Modified broadcast format to remove bracket notation for consistency

FILES MODIFIED:

- server.py (line ~465): Main chat loop in handle_client()
