1. MESSAGE FORMATTING
─────────────────────────────────────────────────────────────────────────────

NEW FORMAT:
  ┌──────────────────────────────┐
  │ 🟢 sender (room)             │
  │ message text here            │
  │ [✓] delivered                │
  └──────────────────────────────┘

Features:
  ✓ Boxed message design (like Discord)
  ✓ Sender indicator (green for own, blue for others)
  ✓ Room name shown
  ✓ Status displayed (delivered/read)
  ✓ Auto-wrapping for long messages
  ✓ Clean separators

4. STATUS DISPLAY
─────────────────────────────────────────────────────────────────────────────

MESSAGE STATUS INDICATORS:

[✓] delivered     ← Message reached the room
                    (shown immediately)

[✓✓] read         ← User acknowledged receipt
                    (shown after /ack command)

Example flow:
  1. alice: hello
     [✓] delivered
  
  2. bob: /ack 1
  
  3. alice sees: hello
                 [✓✓] read

7. FEATURES IMPLEMENTED
─────────────────────────────────────────────────────────────────────────────

UI Classes & Functions:

Colors class:
  • Define ANSI color codes
  • GREEN, BLUE, YELLOW, RED, CYAN, WHITE, GRAY
  • BOLD, RESET for formatting

format_message_ui():
  • Formats chat messages in boxes
  • Shows sender, room, status
  • Auto-wraps long messages
  • Color-coded based on sender

format_system_message():
  • Formats system notifications
  • Uses triangle indicator (▸)
  • Yellow color
  • Clean formatting

parse_and_display_message():
  • Parses server responses
  • Extracts MSG_ID
  • Separates system vs chat
  • Handles wrapping

Message cache:
  • Tracks message_id → status
  • delivered, read states
  • Local client tracking
