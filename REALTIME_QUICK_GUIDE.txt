FEATURES ADDED:
═══════════════════════════════════════════════════════════════════════════════

1. TYPING INDICATORS
   Command: /typing on|off
   Effect: Broadcast "user is typing..." to room
   Room-scoped: Yes (only visible in current room)

2. READ RECEIPTS
   System: Auto message_id assignment
   ACK Command: /ack message_id
   Status: delivered → read tracking

HOW TO USE:
═══════════════════════════════════════════════════════════════════════════════

TYPING INDICATOR:
  alice: /typing on
  bob sees: [SERVER] alice is typing...
  alice: /typing off
  bob sees: [SERVER] alice stopped typing.

READ RECEIPTS:
  alice: hello bob
  MSG_ID:123
  
  bob: /ack 123
  (Message marked as read by bob)

DATA STRUCTURES:
═══════════════════════════════════════════════════════════════════════════════

typing_users = {(socket, room): username}
  → Tracks who is typing in which room
  → Thread-safe with typing_lock

messages = {message_id: {...}}
  → Tracks messages with delivery/read status
  → Auto-cleanup after 1 hour
  → Thread-safe with messages_lock

HELPER FUNCTIONS:
═══════════════════════════════════════════════════════════════════════════════

set_typing(socket, room, username, bool)
  Mark user as typing or not

create_message_with_id(sender, room, text)
  Create message with unique ID

mark_message_delivered(msg_id, username)
  Mark message delivered to user

mark_message_read(msg_id, username)
  Mark message read by user

get_message_status(msg_id)
  Get delivery/read status

COMMANDS ADDED:
═══════════════════════════════════════════════════════════════════════════════
