========================================================
CRITICAL IMPLEMENTATION DETAILS
========================================================

1. BINARY PRESERVATION
   - XOR encryption produces bytes 0-255
   - Some bytes are control chars: \t, \n, \x0b, etc.
   - Solution: Use .lstrip(' ') not .strip()
   - Use split('MSG_ID:') to find message boundary
   - NOT .strip() which removes \t and \x0b

2. DECRYPTION PLACEMENT
   - decrypt() called ONCE per message
   - AFTER message extraction
   - BEFORE display via format_message_ui()
   - Server: does NOT decrypt (forwards encrypted)
   - Client: decrypts immediately before display

3. MESSAGE TYPE ROUTING
   - [SERVER] → system message (not encrypted)
   - [DM from ...] → received DM (encrypted, decrypted)
   - [DM → ...] → sent confirmation (encrypted, decrypted)
   - "user: text" → room message (encrypted, decrypted)
   - Other → silent ignore

4. NO DOUBLE-PROCESSING
   - Each message processed exactly once
   - Each message type has dedicated handler
   - Each handler has explicit return statement
   - No fallback that could cause double-print
