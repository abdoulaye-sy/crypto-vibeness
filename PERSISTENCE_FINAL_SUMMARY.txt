HOW IT WORKS
────────────────────────────────────────────────────────────────────────────

STARTUP FLOW:
1. start_server() called
2. load_message_history() reads chat_history.json
3. message_history dict populated
4. Server ready to accept clients
5. Print: "[HISTORY] Loaded chat history..."

ROOM MESSAGE FLOW:
1. User sends message in room
2. create_message_with_id() creates message ID
3. broadcast_to_room() sends to all users in room
4. save_room_message() adds to message_history
5. save_message_history() writes to JSON file
6. Next server restart → message reloaded

ROOM JOIN FLOW:
1. User executes /join <room>
2. join_room() adds client to room
3. send_room_history_to_client() retrieves last 10 messages
4. Messages displayed: "[SERVER] === ROOM HISTORY ==="
5. New messages continue normally

DM MESSAGE FLOW:
1. User executes /msg <user> <message>
2. handle_dm_command() validates recipient
3. DM sent to recipient socket
4. send_dm() stores in dm_sessions (in-memory)
5. save_dm_message() saves to chat_history.json
6. Next server restart → DM reloaded

────────────────────────────────────────────────────────────────────────────
THREAD SAFETY
────────────────────────────────────────────────────────────────────────────

PROTECTION MECHANISM:
• history_lock = threading.Lock()
• All message_history access wrapped with history_lock

CRITICAL SECTIONS:
1. save_room_message():
   - Lock: Get dict, append message, limit size
   - Call: save_message_history() (lock held during write)

2. save_dm_message():
   - Lock: Get/create session key, append message
   - Call: save_message_history()

3. get_room_history() / get_dm_history():
   - Lock: Read messages, return slice
   - No write operations

4. send_room_history_to_client():
   - Calls: get_room_history() (protected)
   - Thread-safe client send

DEADLOCK PREVENTION:
• history_lock is independent (level 7)
• No other locks held when acquiring history_lock
• Lock released immediately after save
• No nested lock operations
