ARCHITECTURE OVERVIEW
═══════════════════════════════════════════════════════════════════════════════

GLOBAL DATA STRUCTURES:

typing_users = {(socket, room): username}
  Purpose: Track who is currently typing in each room
  Lock: typing_lock
  Memory: ~100 bytes per active typer

messages = {message_id: message_data}
  Purpose: Track all messages with delivery/read status
  Lock: messages_lock
  Auto-cleanup: Deletes after 1 hour
  Memory: ~200 bytes per message

HELPER FUNCTIONS (6 new):

1. set_typing(socket, room, username, typing_bool)
   - Marks user as typing or not in a room

2. get_typing_users(room)
   - Returns list of users typing in a room

3. create_message_with_id(sender, room, text)
   - Creates message with unique ID and tracks it

4. mark_message_delivered(message_id, username)
   - Marks message as delivered to a user

5. mark_message_read(message_id, username)
   - Marks message as read by a user

6. get_message_status(message_id)
   - Gets delivery/read status of a message

COMMANDS (2 new):

1. /typing on
   - Signals user is typing
   - Broadcasts to room: "[SERVER] user is typing..."

2. /typing off
   - Signals user stopped typing
   - Broadcasts to room: "[SERVER] user stopped typing."

3. /ack message_id
   - Acknowledges receipt/read of message
   - Updates message status to read

MESSAGE FLOW DIAGRAM
═══════════════════════════════════════════════════════════════════════════════

1. USER SENDS MESSAGE
   alice: hello
   ↓
2. SERVER CREATES MESSAGE_ID
   message_id_counter incremented → 1
   ↓
3. SERVER STORES MESSAGE
   messages[1] = {
     sender: 'alice',
     room: 'general',
     text: 'hello',
     delivered_to: [],
     read_by: [],
     timestamp: NOW
   }
   ↓
4. SERVER BROADCASTS
   all users in 'general' receive:
   "alice: hello\nMSG_ID:1\n"
   ↓
5. SERVER MARKS DELIVERED
   For each user in room:
   mark_message_delivered(1, username)
   delivered_to: [bob, charlie, ...]
   ↓
6. CLIENTS RECEIVE MESSAGE
   bob sees: "alice: hello\nMSG_ID:1"
   charlie sees: "alice: hello\nMSG_ID:1"
   ↓
7. CLIENT SENDS ACK
   bob: /ack 1
   ↓
8. SERVER MARKS READ
   mark_message_read(1, 'bob')
   read_by: [bob]
   ↓
9. STATUS UPDATE
   Message 1 now shows as read by bob
   (Can be displayed as [✓✓] if client implements it)

THREAD SAFETY IMPLEMENTATION
═══════════════════════════════════════════════════════════════════════════════

NEW LOCKS:

typing_lock
  Protects: typing_users dictionary
  Used by: set_typing(), get_typing_users()
  
messages_lock
  Protects: messages dictionary, message_id_counter
  Used by: create_message_with_id(), mark_message_delivered(),
           mark_message_read(), get_message_status()

LOCK ORDERING (prevents deadlock):
  1. typing_lock
  2. messages_lock
  3. rooms_lock
  4. clients_lock
  5. reset_tokens_lock

ATOMIC OPERATIONS:

set_typing():
  with typing_lock:
    typing_users[(socket, room)] = username

create_message_with_id():
  with messages_lock:
    message_id_counter += 1
    messages[msg_id] = {...}

mark_message_delivered():
  with messages_lock:
    messages[msg_id]['delivered_to'].append(username)

MEMORY MANAGEMENT
═══════════════════════════════════════════════════════════════════════════════

TYPING USERS:
  - ~100 bytes per active typer
  - Cleared immediately on /typing off
  - Cleared on disconnect
  - No memory buildup

MESSAGES:
  - ~200 bytes per message
  - Auto-deleted after 1 hour
  - cleanup_old_messages() removes expired
  - Prevents unbounded memory growth

EXAMPLE: 100 concurrent users, 5000 messages/hour
  - Typing: 100 × 100 bytes = 10 KB
  - Messages (1-hour buffer): ~1 MB
  - Total: ~1 MB per hour
  - Well within limits for typical systems
