WHAT WAS REQUESTED
════════════════════════════════════════════════════════════════════════════════

Ajoute un système de messages privés (DM) au chat existant.

OBJECTIF: Permettre aux utilisateurs de s'envoyer des messages privés hors des
rooms.

========================
1. NOUVELLE COMMANDE
========================

/msg <username> <message>

Exemple :
/msg vincent salut bro

========================
2. COMPORTEMENT
========================

- le message ne doit PAS être broadcast dans une room
- il doit être envoyé uniquement au destinataire
- le destinataire reçoit :

[DM from <sender>]: <message>

- l'expéditeur voit :

[DM → <recipient>]: <message>

========================
3. GESTION DES SESSIONS
========================

Créer une structure en mémoire :

dm_sessions = {
  "user1:user2": [messages]
}

clé ordonnée (alphabetical order) pour éviter duplication

========================
4. CONTRAINTES
========================

- ne pas casser rooms system
- ne pas casser auth/login
- ne pas modifier client UI sauf affichage DM
- server.py modification principale
- thread-safe obligatoire

ARCHITECTURE:
─────────────

dm_sessions dictionary:
- Key: "user1:user2" (alphabetically ordered for consistency)
- Value: List of message dicts
- Thread-safe access via dm_lock

Example:
  dm_sessions = {
      "alice:bob": [
          {'sender': 'alice', 'text': 'hello', 'timestamp': ...},
          {'sender': 'bob', 'text': 'hi', 'timestamp': ...}
      ]
  }

PROTOCOL:
─────────

Successful message flow:
1. User A types: /msg B hello
2. Server parses command
3. Server checks if B exists in database
4. Server checks if B's socket is connected
5. Server sends "[DM from A]: hello" to B's socket
6. Server sends "[DM → B]: hello" to A's socket
7. Message stored in dm_sessions["a:b"]
8. Both users see formatted messages

Error handling:
1. Usage error: /msg (no args)
2. Empty message: /msg B (no message)
3. User not found: /msg nouser hello
4. User offline: /msg offline hello (user in database but not connected)
5. Send failure: Socket error during transmission

THREAD SAFETY:
───────────────

dm_lock protects: dm_sessions dictionary
  - Used in send_dm() function
  - Critical section: ~5 lines
  - Low contention (only written at message send)

clients_lock protects: clients dictionary
  - Used in find_recipient_socket() function
  - Correct lock hierarchy (already used by other functions)
  - No deadlock risk

Lock ordering verified:
  typing_lock (1) → messages_lock (2) → rooms_lock (3) →
  clients_lock (4) → reset_tokens_lock (5)
  
  find_recipient_socket uses clients_lock (level 4) - CORRECT
